# 집중 여행 서비스 홈 UX 재설계

작성일: 2026-03-29

## 1. 재설계 방향

- 홈 화면은 토스처럼 명확하고 실용적인 구조를 가진다.
- 동시에 싸이월드 미니룸처럼 감성적인 장면 경험을 제공한다.
- 사용자는 기능 카드를 먼저 보는 대신, 하나의 여행 장면 안에 들어온 느낌을 먼저 받는다.
- 홈의 역할은 동행 선택과 출발 진입까지로 제한한다.
- 시간 선택, 세부 설정, 보조 기능은 홈 밖으로 분리한다.

## 2. 핵심 콘셉트

- 홈 화면은 공항 미니룸 콘셉트로 설계한다.
- 여러 아바타가 같은 공간 안에 동시에 존재한다.
- 각 아바타는 공항 안에서 서로 다른 생활 행동을 수행한다.
- 사용자는 화면 속 아바타를 직접 탭해 동행을 고른다.

예시 행동:

- 캐리어를 끌고 이동
- 의자에 앉아 간식 먹기
- 기대서 잠들기
- 휴대폰 보기
- 창밖이나 전광판 보기

## 3. 전체 정보 구조

화면은 위에서 아래로 다음 흐름을 가진다.

1. 공항 전광판형 상단 문구
2. 공항 미니룸 장면과 여러 아바타
3. 선택 상태와 출발 CTA가 있는 하단 반투명 바
4. 고정형 하단 GNB

각 영역은 분리된 카드처럼 보이지 않고, 하나의 장면 위에 자연스럽게 얹힌 인터페이스처럼 이어져야 한다.

## 4. 상단 문구 영역

- 일반 반투명 패널 대신 공항 전광판형 오브제를 사용한다.
- 짙은 남색 또는 코발트 블루 계열 배경을 사용한다.
- 공항 출발 안내판처럼 공간 안의 실제 사물로 읽혀야 한다.
- 문구는 2줄 이하로 짧게 유지한다.
- 역할은 설명이 아니라 여행 세계관 진입을 돕는 입구다.

예시 구성:

- 상단 작은 라벨: `BOARDING NOW`
- 메인 문구: `오늘은 누구와 떠나볼까요?`

## 5. 중앙 미니룸 영역

공항 장면은 다음 3개 구역으로 구성한다.

- 상단: 전광판, 창밖 활주로, 게이트 배경
- 중단: 좌석 대기 구역과 이동 통로
- 하단: 카페형 휴식 구역 또는 탑승 대기 구역

아바타는 랜덤 나열이 아니라 공항 동선에 맞는 위치에 배치한다. 이로써 장면은 예쁜 일러스트가 아니라 실제로 고를 수 있는 생활 공간처럼 보이게 된다.

## 6. 아바타 선택 인터랙션

- 한 화면에 여러 아바타가 동시에 보인다.
- 사용자는 장면 안의 아바타를 직접 탭해 동행을 선택한다.
- 선택된 아바타는 아래 이름 라벨, 은은한 글로우, 약간 높은 선명도로 강조한다.
- 다른 아바타보다 선택된 아바타 한 명만 또렷하게 읽혀야 한다.
- 선택 즉시 하단 CTA 문구가 선택된 인물 기준으로 바뀐다.

## 7. 하단 시작 영역

- GNB 바로 위에 밝은 반투명 바를 둔다.
- 이 영역은 선택 요약과 하나의 CTA만 보여준다.
- 여러 정보나 설정 항목은 올리지 않는다.
- 역할은 사용자가 선택한 동행을 자연스럽게 다음 단계로 넘기는 연결부다.

### 7-1. 선택 상태가 있을 때

- 문구는 1줄만 사용한다.
- 예시: `빠와 함께 떠나기`
- CTA는 중앙 하단 전체폭 버튼 1개만 둔다.

### 7-2. 선택 상태가 없을 때

- 상태 문구: `함께 여행할 동행을 골라주세요`
- 버튼 문구: `동행을 선택해주세요`

## 8. 하단 GNB

- 홈 화면의 보조 기능은 전부 GNB로 분리한다.
- GNB는 화면 하단에 떠 있는 밝은 반투명 캡슐 바 형태로 설계한다.
- CTA 바와 재질감은 맞추되, 더 안정적이고 고정된 느낌을 준다.
- 배경을 완전히 가리기보다, 뒤 장면이 아주 은은하게 비치게 한다.

메뉴 구성:

- 홈
- 기록
- 소리
- 메뉴

### 8-1. 아이콘과 텍스트 톤

- 아이콘은 도트형이 아니라 단순한 심볼형으로 사용한다.
- 라벨은 반드시 함께 노출한다.
- 선택된 메뉴만 포인트 컬러를 사용하고, 나머지는 차분한 회색으로 처리한다.
- 활성 상태는 과장된 배경 칩보다 아이콘과 텍스트의 선명도로 구분한다.

권장 톤:

- 배경: 밝은 반투명 화이트
- 활성 포인트: 코랄 계열
- 비활성: 블루그레이 또는 중간 회색

## 9. UX 원칙

- 홈에서는 동행 선택과 출발 진입만 처리한다.
- 시간 선택은 CTA를 누른 다음 화면에서 진행한다.
- 화면 안의 주 행동은 항상 하나만 유지한다.
- 감성은 배경, 장면, 움직임에서 만들고 UI는 짧고 명확하게 유지한다.
- 상단, 중앙, 하단, 최하단은 역할이 달라도 시각적으로 끊기지 않아야 한다.

## 10. 다음 설계 대상

이 홈 구조를 기준으로 다음 단계에서 아래 항목을 이어서 구체화한다.

- 홈에서 시간 선택 화면으로 넘어가는 전환 UX
- 시간 선택 화면의 톤과 정보 구조
- 선택한 동행과 여행 공간이 다음 화면에서 어떻게 이어지는지에 대한 연속성
