# 집중 여행 서비스 디자인 스펙

작성일: 2026-03-20

## 1. 서비스 한 줄 정의

여행 서사를 배경으로, 화이트노이즈와 함께 집중을 돕는 몰입형 서비스.

이 서비스의 본질은 여행 감상이 아니라 사용자의 집중 유지다. 여행 연출은 집중을 더 오래 지속하게 만드는 정서적 장치로 작동해야 한다.

## 2. 제품 목표

- 사용자가 차분하게 집중에 진입하도록 돕는다.
- 화이트노이즈를 자연스럽게 제공한다.
- 여행 크리에이터 감성의 세계관을 통해 기존 집중 앱과 다른 몰입감을 만든다.

## 3. 핵심 원칙

- 집중 효용이 가장 우선이다.
- 화면은 화려하기보다 조용하고 심플해야 한다.
- 사용자는 화면을 오래 보지 않으므로, 과한 전환이나 정보 노출을 피한다.
- 여행 연출은 기능을 설명하는 장식이 아니라 집중을 받쳐주는 분위기 장치여야 한다.
- 핀터레스트 감성은 직접적인 스타일 복제가 아니라, 따뜻하고 정적인 무드로 번역한다.

## 4. 핵심 사용자 경험

사용자는 앱에 들어와 여러 여행 크리에이터 감성의 캐릭터 중 하나를 선택한다. 그다음 얼마나 여행을 떠날지, 즉 집중 시간을 고른다. 세션이 시작되면 선택한 캐릭터의 여행 스타일에 맞춰 국가와 코스가 랜덤으로 정해지고, 공간별 화이트노이즈가 자동 재생된다. 사용자는 여행 서사를 느슨하게 배경으로 두고 집중을 이어간다.

핵심 UX 흐름은 아래와 같다.

1. 유튜버 오마주형 캐릭터 선택
2. 집중 시간 선택
3. 여행 세션 시작
4. 공간에 맞는 화이트노이즈 자동 재생
5. 조용한 장면 변화 속 집중 유지
6. 도착형 종료 경험 제공

## 5. 첫 화면 UX

첫 화면은 감성적인 출발 전 공간으로 구성한다. 이 공간 안에는 여러 명의 유튜버 오마주형 도트 3D 아바타가 함께 존재한다. 사용자는 목록형 카드 UI를 고르는 느낌보다, 하나의 세계 안에서 자신이 따라가고 싶은 캐릭터를 직접 선택하는 느낌을 받아야 한다.

첫 화면의 역할은 기능 설명이 아니라 서비스 세계관 진입이다.

- 여러 캐릭터가 한 공간 안에 자연스럽게 배치된다.
- 사용자는 원하는 캐릭터를 직접 선택한다.
- 선택 순간 특정 여행 세계관에 입장하는 감정을 준다.
- 이후 바로 집중 시간 선택 단계로 이동한다.

사운드는 첫 진입 단계에서 직접 선택하지 않는다. 기본 사운드는 코스와 공간에 따라 자동 재생되며, 사용자는 세션 중 필요할 때만 사운드 변경 옵션을 사용할 수 있다.

## 6. 세션 경험

세션 화면은 시네마형 구조를 따른다.

- 화면의 대부분은 도트형 여행 공간과 아바타가 차지한다.
- 타이머와 사운드 UI는 작고 조용하게 배치한다.
- 사용자는 화면을 오래 보지 않기 때문에, 정보 UI는 최소화한다.

여행 공간 예시는 다음과 같다.

- 비행기
- 공항 라운지
- 카페
- 음식점
- 숙소
- 액티비티 공간
- 관광지

세션의 핵심은 공간이 빠르게 바뀌는 것이 아니라, 한 공간 안에서 차분한 체류감을 주는 것이다.

## 7. 공간과 행동 연출 원칙

한 세션은 여러 개의 여행 공간으로 구성된다. 하지만 각 공간 안에서는 배경이 크게 변하지 않는다. 변화는 주로 아바타의 행동으로만 표현한다.

예를 들어 비행기 공간에서는 다음과 같은 방식이 가능하다.

- 배경은 같은 좌석 공간으로 유지
- 아바타는 앉아 있기
- 아바타는 잠자기
- 아바타는 식사하기
- 아바타는 독서하기

즉, 한 공간 안에서는 배경이 안정적으로 유지되고, 아바타 행동만 2~3회 정도 조용하게 변화한다. 공간 전환 자체는 적고, 내부 행동 변화가 몰입을 만든다.

## 8. 콘텐츠 구조

첫 버전은 5명의 소수 정예 캐릭터로 시작한다. 초기에는 실존 유튜버 실명이나 실사 재현이 아니라, 여행 유튜버 감성에서 영감을 받은 오마주형 캐릭터로 운영한다. 추후 실제 협업이나 제휴가 붙으면 공식 크리에이터 콘텐츠로 확장할 수 있다.

각 캐릭터는 단순히 외형이 다른 것이 아니라, 고유한 여행 스타일 프로필을 가진다. 여행 스타일 프로필은 공간 유형별 점유 비율로 설계한다.

예시:

- 비행기: 10%
- 카페: 5%
- 음식점: 20%
- 숙소: 20%
- 액티비티: 45%

사용자가 총 집중 시간을 선택하면, 시스템은 이 비율에 맞춰 공간별 체류 시간을 자동 배분한다. 따라서 같은 30분 세션이라도 어떤 캐릭터를 고르느냐에 따라 여행의 리듬이 달라진다.

## 9. 코스 생성 방식

- 사용자는 집중 시간을 선택한다.
- 시스템은 선택된 캐릭터의 여행 스타일 프로필을 불러온다.
- 시스템은 해당 캐릭터가 가진 여행 국가와 코스 중 하나를 랜덤 선택한다.
- 전체 시간에 맞춰 공간별 체류 시간을 배분한다.
- 각 공간은 배경 고정 + 아바타 행동 변화 구조로 재생된다.
- 공간에 맞는 기본 화이트노이즈가 자동 재생된다.

이때 랜덤성은 국가와 코스에서 오고, 캐릭터다움은 여행 스타일 프로필에서 온다.

## 10. 사운드 정책

사운드는 사용자가 진입 시점에 먼저 고르는 요소가 아니다. 기본값은 여행 코스와 현재 공간에 따라 자동 결정된다.

예시:

- 비행기 공간: 기내 소음
- 비 오는 숙소: 빗소리
- 카페 공간: 잔잔한 카페 앰비언스
- 이동 공간: 교통수단 기반 환경음

다만 사용자는 세션 중 언제든 다른 화이트노이즈로 변경할 수 있어야 한다. 즉 기본 경험은 자동 연출이고, 개인화는 세션 안에서 보조적으로 제공한다.

## 11. 세션 종료 경험

세션 종료는 단순 타이머 종료가 아니라, 여행의 도착 감정을 줘야 한다.

- 기본은 도착형 종료다.
- 짧은 문구로 여행이 끝났음을 알린다.
- 그 아래에 기록형 요약을 짧게 보여준다.

요약 정보 예시:

- 오늘의 집중 시간
- 따라간 여행 루트
- 선택한 캐릭터

종료 경험의 목적은 성취감을 주되, 과하게 떠들지 않는 것이다.

## 12. MVP 범위

첫 버전에서 반드시 필요한 요소는 아래와 같다.

- 오마주형 캐릭터 5종
- 캐릭터별 여행 스타일 프로필
- 집중 시간 선택
- 랜덤 국가 및 코스 선택
- 도트형 여행 공간 연출
- 공간별 기본 화이트노이즈 자동 재생
- 세션 중 사운드 변경 옵션
- 도착형 종료 + 짧은 기록 요약

첫 버전에서 과감히 미루는 요소는 아래와 같다.

- 실제 유튜버 협업
- 복잡한 커뮤니티 기능
- 소셜 공유 중심 기능
- 과도한 커스터마이징
- 게임화가 강한 보상 시스템

## 13. 향후 확장 방향

- 실제 크리에이터 제휴 캐릭터 및 코스 출시
- 캐릭터별 테마팩 판매
- 시즌성 여행 패스 출시
- 집중 기록 기반 개인 추천
- 프리미엄 화이트노이즈 패키지

## 14. 요약

이 서비스는 여행을 보여주는 앱이 아니라, 여행의 리듬을 빌려 집중을 돕는 앱이다. 사용자는 좋아하는 여행 크리에이터 감성의 캐릭터를 선택하고, 그 캐릭터의 여행 스타일을 따라 조용한 공간과 화이트노이즈 속에서 집중한다. 화면은 심플해야 하고, 공간은 안정적이어야 하며, 변화는 아바타의 느린 행동을 중심으로 일어나야 한다.

## 15. 화면별 상세 기획

### 15-1. 첫 진입 화면

- 배경은 여행 전날의 감성적인 자취방이다.
- 같은 공간 안에 5명의 유튜버 오마주형 도트 3D 캐릭터가 함께 존재한다.
- 사용자는 화면 속 캐릭터를 직접 탭해 선택한다.
- 캐릭터 배치는 군집형으로 구성해 하나의 세계 안에 들어온 느낌을 준다.
- 이 화면은 리스트 탐색보다 특정 여행 세계관에 입장하는 경험을 우선한다.

고정 카피:

- 메인: 누구와 여행을 떠날까요?
- 서브: 함께 여행할 친구를 선택해주세요

### 15-2. 출발 전환 화면

- 캐릭터 선택 후, 공항 출발 라운지 느낌의 짧은 전환 장면을 보여준다.
- 자취방에서 공항으로 이어지는 흐름을 통해 실제 출발감을 강화한다.
- 이 장면은 길지 않게 유지하고, 다음 단계인 시간 선택으로 자연스럽게 넘긴다.

### 15-3. 집중 시간 선택 화면

- 사용자는 타이머를 설정하는 느낌이 아니라 여행 거리를 고르는 느낌을 받아야 한다.
- 시간 표현은 감성 문구와 실제 시간을 함께 보여주는 혼합형을 사용한다.
- 첫 버전은 짧은 시간 없이 1시간부터 시작한다.

시간 선택지:

- 국내여행 (1시간)
- 아시아 여행 (2시간)
- 유럽 여행 (3시간)
- 미국 여행 (4시간)
- 남극 여행 (5시간)

고정 카피:

- 메인: 어디로 떠날까요?
- CTA: 떠나볼까요?

### 15-4. 세션 진행 화면

- 세션 화면은 시네마형으로 설계한다.
- 화면 대부분은 도트형 3D 여행 공간과 아바타가 차지한다.
- 타이머와 사운드 UI는 좌상단 작은 오버레이로 배치한다.
- 사용자는 필요할 때만 시간을 힐끗 확인할 수 있어야 한다.
- 현재 위치 정보는 항상 노출하지 않고, 공간이 바뀔 때만 잠깐 보여준다.

위치 정보 표기 방식:

- 실제 여행지 중심으로 표현한다.
- 예시: 다낭으로 향하는 비행기 안, 다낭의 작은 카페, 오늘의 숙소

### 15-5. 공간 연출 방식

- 한 세션은 여러 개의 여행 공간으로 구성된다.
- 각 공간에서는 배경이 크게 바뀌지 않고 안정적으로 유지된다.
- 변화는 아바타의 느린 행동 변화 중심으로 일어난다.
- 한 공간 안에서 아바타는 2~3가지 행동만 수행한다.

예시:

- 비행기 공간: 앉아 있기, 잠자기, 식사하기, 독서하기
- 숙소 공간: 창밖 보기, 침대에 눕기, 짐 정리하기
- 카페 공간: 음료 마시기, 창밖 보기, 가만히 앉아 있기

### 15-6. 사운드 변경 패널

- 사운드 변경은 세션 중 필요할 때만 여는 보조 기능이다.
- 진입 방식은 좌상단 오버레이의 사운드 버튼 탭이다.
- 노출 방식은 화면 하단에서 올라오는 작은 바텀시트형이다.
- 기본값은 현재 공간에 맞는 자동 화이트노이즈다.
- 사용자는 필요 시 다른 화이트노이즈로 변경할 수 있다.

### 15-7. 세션 종료 화면

- 종료 화면은 살짝 감성적인 도착형 톤을 따른다.
- 과한 축하보다 여행을 잘 마쳤다는 여운을 주는 것이 중요하다.
- 짧은 도착 문구 아래에 기록형 요약을 덧붙인다.

표시 정보 예시:

- 오늘의 집중 시간
- 따라간 여행 루트
- 선택한 캐릭터
